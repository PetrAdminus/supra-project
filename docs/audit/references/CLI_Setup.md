# Справочник: Supra CLI — установка и частые проблемы

## 1. Ошибка при установке: запрос значения `Uri`
**Симптомы.** PowerShell запрашивает параметр `Uri` при выполнении `Invoke-WebRequest` во время установки CLI.

**Решение.** Используйте Git Bash или иной терминал, который корректно обрабатывает команду скачивания. Это позволяет избежать ручного ввода значения URI и завершить установку без вмешательства.

## 2. Публикация модуля: `MODULE_ADDRESS_DOES_NOT_MATCH_SENDER`
**Симптомы.** Деплой останавливается с сообщением `"vm status": "MODULE_ADDRESS_DOES_NOT_MATCH_SENDER"`.

**Решение.**
1. Убедитесь, что используемый CLI-профиль действительно создан и активирован.
2. Выполните `supra config show-profiles` и убедитесь, что профиль со звёздочкой совпадает с именованным адресом в `Move.toml`.
3. Повторите публикацию только после синхронизации адресов.

## 3. Команда `supra` не запускается
**Симптомы.** Вызов `supra` не приводит к запуску CLI.

**Решение.**
1. Проверьте, что алиасы Docker настроены корректно.
2. Убедитесь, что контейнер `supra_cli` запущен через Docker Desktop или командой `docker ps`.
3. При необходимости перезапустите контейнер и повторите команду.

## 4. Ошибка Docker: демон не запущен
**Симптомы.** Команда запуска контейнера возвращает ошибку о неработающем Docker Daemon.

**Решение.**
1. Запустите Docker Desktop перед выполнением команды `docker run`.
2. Повторите запуск контейнера после проверки статуса службы Docker.
3. Если ошибка повторяется, перепроверьте корректность установки Docker и автозапуск сервиса.

## 5. Аналог `create-resource-account` в Supra CLI
**Симптомы.** Требуется создать ресурсный аккаунт и опубликовать пакет через CLI.

**Решение.** Нативной команды пока нет; вызовите функцию `create_resource_account` вручную из Move-модуля. В качестве примера можно использовать реализацию из репозитория Supra Framework.
