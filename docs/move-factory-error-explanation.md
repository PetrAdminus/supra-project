# Ошибки компиляции пакета `lottery_factory`

Логи GitHub Actions показывают, что после успешной сборки пакета `lottery` происходит запуск `aptos move compile` для пакета `lottery_factory`. Компилятор Move выдаёт три ошибки:

1. `entry function cannot return values` — входная функция, которую можно вызывать транзакцией, обязана иметь тип возврата `()` (ничего не возвращать). В файле `LotteryFactory.move` функция `create_lottery` помечена как `public entry fun`, но при этом возвращает значение, поэтому компилятор отвергает её сигнатуру.
2. Две ошибки `type ... is not supported as a transaction parameter type` — параметры транзакции могут быть только примитивами, адресами, векторам из примитивов или ссылками на ресурсы, допускаемые стандартом. Пользовательский структурный тип `LotteryBlueprint` не соответствует этим ограничениям, из-за чего передача значения этого типа в `entry`-функции запрещена. Компилятор сообщает об этом как об «unsupported transaction parameter type» для аргументов `blueprint` в функциях `create_lottery` и `update_blueprint`.

Пока сигнатуры `entry`-функций не будут изменены (убрать возвращаемое значение и заменить параметры на допустимые типы либо оборачивать логику в отдельные `public`-функции), компиляция пакета `lottery_factory` будет завершаться с ошибкой, и GitHub Actions не пройдет.
